Когда мы пишем в строке браузера запрос, его принимает Dispatcher Servlet,
далее он находит для обработки этого запроса подходящий контроллер с помощью
HandlerMapping (это такой интерфейс для выбора контроллера, проверяет в каком из
имеющихся контроллеров есть метод, принимающий такой адрес),
вызывается подходящий метод и Controller возвращает информацию о представлении,
затем диспетчер находит нужное представления по имени при помощи ViewResolver'а,
после чего на это представление передаются данные модели и на выход мы получаем нашу страничку.
 Как-то так.
